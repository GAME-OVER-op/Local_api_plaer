# Планшет Плеер + media-server

Проект из двух частей:

- **app/** — Android-приложение (плеер на libVLC) для старого планшета (Android 4.4.2, API 19, armeabi-v7a).
- **server/** — Rust-бинарник `media-server` для второго (root) устройства (aarch64), отдаёт файлы по локальной сети.
- **magisk/** — шаблон Magisk-модуля для автозапуска сервера без монтирования в систему.

## Сервер (media-server)

Запуск вручную на root-устройстве:

```
./media-server -H 0.0.0.0 -p 10930 -r /storage -n "Гостиная"
```

Параметры (также через переменные окружения MEDIA_HOST/MEDIA_PORT/MEDIA_ROOT/MEDIA_NAME/MEDIA_BACKGROUND/MEDIA_DATA_DIR):

| Флаг | Описание | По умолчанию |
|------|----------|--------------|
| `-H, --host` | адрес прослушивания | `0.0.0.0` |
| `-p, --port` | порт (TCP и UDP) | `10930` |
| `-r, --root` | корневая папка | `/storage` |
| `-n, --name` | имя сервера для автообнаружения | `media-server` |
| `-b, --background` | фоновый режим (без консоли) | выкл. |
| `-d, --data` | папка данных (allowed.json / access.log) | рядом с бинарником |

Корневая папка по умолчанию — `/storage`, чтобы были видны и внутренняя память (`/storage/emulated/0`), и карта памяти (`/storage/####-####`). Файлы данных (список разрешённых и лог) хранятся рядом с бинарником — в Magisk-модуле это папка модуля.

### Маскировка и защита

Сервер не раскрывает ни модель устройства, ни назначение. Для любого «чужого» запроса (не от одобренного планшета) он отвечает как обычный веб-сервер-заглушка:

- заголовок `Server: nginx/1.18.0 (Ubuntu)` и типовая HTML-страница nginx;
- никакой технической информации о планшете или Android не выдаётся;
- настоящие эндпоинты (`/list`, `/search`, `/download`) для них невидимы — только заглушка.

Автообнаружение по UDP при этом работает — планшет найдёт сервер в сети (ответ только на специальный широковещательный запрос `MEDIA_DISCOVER`, без деталей об устройстве).

При первом обращении нового устройства сервер спрашивает в консоли:

```
=== Запрос доступа ===
Устройство: "Samsung SM-T230"  (IP 192.168.0.55, id a1b2c3…)
Разрешить? [y/n] (30 сек):
```

- `y` — разрешить, `n` — отклонить. Нет ответа 30 сек — автоматический отказ.
- Разрешённые устройства сохраняются в `allowed.json` (в папке данных) и больше не спрашиваются.
- Идентификатор — ANDROID_ID планшета + его название.
- В фоновом режиме (`-b`, как в Magisk) пускаются только устройства из `allowed.json`, остальным — заглушка.
- Попытки чужих запросов пишутся в `access.log`.

Команды консоли во время работы: `list` — список разрешённых, `revoke <id|имя>` — отозвать, `help` — справка.

### API (только для одобренных устройств)

- `GET /list?path=<отн>` — содержимое папки (JSON).
- `GET /search?q=<текст>[&path=<отн>]` — рекурсивный поиск (до 500).
- `GET /download?path=<отн>` — отдача файла с поддержкой Range (перемотка/докачка).

Идентификатор устройства передаётся заголовками `X-Device-Id`/`X-Device-Name` либо query-параметрами `dev`/`dn` (для потока libVLC).

## Magisk-модуль (автозапуск)

Папка `magisk/` — шаблон модуля (`id=api_plaer`). CI собирает из него готовый zip с вложенным бинарником.

- **Не монтируется в систему** — только запускает сервер через `service.sh` после загрузки.
- Бинарник, `allowed.json`, `access.log` и `config.env` лежат в папке модуля `/data/adb/modules/api_plaer/`.
- Настройки запуска — в `config.env` (MEDIA_HOST/PORT/ROOT/NAME/BACKGROUND).
- При обновлении модуля `customize.sh` сохраняет `config.env` и `allowed.json` (список разрешённых не теряется; `access.log` может быть перезаписан).

> Первый раз разрешить планшет нужно интерактивно: запустите бинарник вручную через `su` (без `-b`), подтвердите доступ `y` — после этого фоновый автозапуск будет пускать его автоматически.

## Приложение

- Главный экран: IP и порт, кнопка **Найти серверы в сети** (автообнаружение по UDP), список недавних подключений. Клавиатура не всплывает при запуске.
- Проводник: поиск, сортировка по имени (естественная, Серия 2 раньше Серии 10), потянуть вниз — обновить, отметка «✓» на просмотренных, метка «⏱» — есть место остановки, «⬇» — уже скачано.
- Навигация по папкам — «хлебные крошки» (путь с кликабельными сегментами) вверху.
- **Кнопка «Загрузки»** на панели проводника (как в старом Chrome) открывает отдельный экран с уже скачанными файлами (просмотр/удаление только своих загрузок).
- Скачивание через foreground-сервис с прогрессом в шторке; сразу после запуска — плашка «ОТМЕНИТЬ» (на случай случайного нажатия), отмена также из уведомления.
- APK можно скачать и сразу предложить установку («Скачать и установить») — для обновления самого плеера на старом планшете.
- Плеер: имя файла сверху по центру при показе управления, процент подготовки («Подготовка… NN%»), продолжение с места остановки, увеличенный буфер (меньше артефактов), жесты (двойной тап ±10 сек / пауза, горизонтальное перетаскивание — перемотка, вертикальное справа — громкость до 200%), скорость, соотношение сторон, полноэкранный режим.
- Кнопки «предыдущая/следующая серия»; по окончании — окно со списком остальных серий и автопереходом.
- **При обрыве сервера плеер не выходит**: показывает статус «Поиск сервера…», переобнаруживает сервер по UDP (даже если сменился IP) и продолжает тот же файл с той же позиции.
- Настройки: светлая/тёмная тема.

## Сборка

Сборка автоматически в GitHub Actions (`.github/workflows/build.yml`):

- `server` — собирает `media-server` (aarch64) и Magisk-модуль. Артефакты: `media-server-aarch64`, `media-server-magisk`.
- `server-selftest` — проверяет, что чужим отдаётся ровно один заголовок `Server: nginx` (маскировка без дубликатов).
- `app` — собирает APK. Артефакт: `tablet-player-apk`.
