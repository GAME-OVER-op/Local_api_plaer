# Планшет-плеер — media-server + Android-приложение

Превращает старый Android-планшет в сетевой видеоплеер. Проект состоит из двух частей:

1. **`server/`** — лёгкий многопоточный HTTP-сервер на Rust. Запускается на **рутованном** Android-устройстве, читает каталог (по умолчанию `/storage/emulated/0`) и отдаёт файлы по локальной сети.
2. **`app/`** — Android-приложение (плеер) на базе **libVLC**. Спрашивает IP и порт сервера, показывает список файлов, воспроизводит видео (с перемоткой) и скачивает файлы на планшет.

## Целевые платформы

| Компонент | Платформа | Примечание |
|----------|----------|-----------|
| Приложение | Android 4.4.2 (API 19), ABI `armeabi-v7a` | `minSdk 19`, `targetSdk 22`. libVLC 3.x требует минимум API 17. |
| Сервер | Android, ARM64 (`aarch64-linux-android`) | Запуск требует root для полного доступа к хранилищу. |

> ⚠ Сеть считается **доверенной**: авторизации нет. Не выставляйте сервер в интернет.

## Сервер (`server/`)

### Запуск

Скопируйте собранный бинарник `media-server` на устройство (например в `/data/local/tmp`) и запустите из root-оболочки:

```sh
adb push media-server /data/local/tmp/
adb shell
su
chmod 755 /data/local/tmp/media-server
/data/local/tmp/media-server            # дефолт: 0.0.0.0:10930, /storage/emulated/0
```

Параметры:

```
-H, --host <HOST>   адрес прослушивания (по умолчанию 0.0.0.0)
-p, --port <PORT>   порт (по умолчанию 10930)
-r, --root <PATH>   корневой каталог (по умолчанию /storage/emulated/0)
-h, --help          справка
```

Те же значения можно задать через переменные окружения `MEDIA_HOST`, `MEDIA_PORT`, `MEDIA_ROOT`.

### Многопоточность (важно для воспроизведения)

Сервер обрабатывает запросы пулом воркеров (по числу ядер, минимум 4). Это принципиально: при воспроизведении libVLC держит несколько соединений одновременно (поток данных + отдельные `Range`-запросы на перемотку). Прежний однопоточный цикл блокировался на первом соединении — видео «висело на 0». Каждый запрос логируется строкой вида `Get /download?path=... -> 206`.

### API

| Метод | Путь | Описание |
|-------|------|-----------|
| GET | `/` или `/health` | Проверка живости (`media-server ok`). |
| GET | `/list?path=REL` | Список содержимого папки (JSON). `path` — относительно корня, может быть пустым. |
| GET | `/search?q=TEXT` | Глобальный рекурсивный поиск по имени. Необязательный `&path=REL` — ограничить поддеревом. |
| GET | `/download?path=REL` | Отдача файла. Поддерживает заголовок `Range` (перемотка и докачка). |

Поиск обходит дерево от корня, пропускает символические ссылки, ограничен 500 результатами. Папки выводятся первыми. Пути ограничены корневым каталогом (защита от path traversal).

## Приложение (`app/`)

### Функции

- Экран подключения: ввод IP и порта (сохраняются) + **история подключений** (тап по записи — быстрый повторный вход).
- Файловый браузер с навигацией по папкам и плавными анимациями переходов.
- **Глобальный поиск** по всем файлам (через `/search`): результаты показывают расположение и открываются/воспроизводятся напрямую. Кнопка «Назад» выходит из режима поиска.
- Тап по файлу: «Смотреть» (плеер) или «Скачать». Долгое нажатие — сразу скачать.
- Скачивание файлов в папку `Download` планшета с индикатором прогресса.

### Плеер (libVLC)

- Пауза/воспроизведение, перемотка ±10с, отдельная кнопка **+1:30**, ползунок прогресса.
- **Перемотка жестом**: горизонтальный свайп перематывает пропорционально длине жеста (весь экран по ширине ≈ 2 минуты); по ходу показывается смещение и целевое время, перемотка применяется при отпускании.
- **Одиночный тап** по видео — показать/скрыть панель управления (автоскрытие через 3.5с).
- **Соотношение сторон**: цикл По размеру → 16:9 → 4:3 → Растянуть → Оригинал.
- Выбор скорости (0.5x–2x).
- **Экран не гаснет** во время воспроизведения (`FLAG_KEEP_SCREEN_ON`).

### Технические заметки

- Воспроизведение идёт **потоком** напрямую с `GET /download?path=...` — libVLC сам тянет данные и использует `Range` для перемотки.
- `targetSdk 22` сознательно: на API 19 разрешения выдаются при установке, а обычный HTTP (cleartext) работает без доп. настроек.
- Приложение не использует AppCompat — только штатные темы Holo, чтобы не тянуть AndroidX.

## CI (GitHub Actions)

Файл `.github/workflows/build.yml` собирает оба артефакта параллельно:

| Job | Что делает | Артефакт |
|-----|-----------|----------|
| `server` | Кросс-компиляция Rust под `aarch64-linux-android` через `cargo-ndk` | `media-server-aarch64` (бинарник) |
| `app` | Сборка `assembleRelease` с R8 (сжатие/оптимизация) и подписью debug-ключом | `tablet-player-apk` (`app-release.apk`) |

Артефакты доступны во вкладке Actions. Запуск — по push в `main`/`master` или вручную (workflow_dispatch).

### Почему release + R8, но debug-подпись

Релизная сборка включает R8 (`minifyEnabled` + `shrinkResources`) — меньше размер и быстрее код, но подписывается штатным debug-ключом, чтобы готовый `app-release.apk` ставился без собственного keystore. Правила R8 для libVLC (JNI) — в `app/app/proguard-rules.pro`.

## Ограничения

- Нет авторизации — только для доверенной локальной сети.
- APK подписан debug-ключом — для личной установки этого достаточно.
- Серверу нужен root, если часть каталогов недоступна обычному пользователю.
