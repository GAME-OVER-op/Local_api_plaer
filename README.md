# Планшет Плеер + media-server

Проект из двух частей:

- **app/** — Android-приложение (плеер на libVLC) для старого планшета (Android 4.4.2, API 19, armeabi-v7a).
- **server/** — Rust-бинарник `media-server` для второго (root) устройства (aarch64), отдаёт файлы по локальной сети.

## Сервер (media-server)

Запуск вручную на root-устройстве:

```
./media-server -H 0.0.0.0 -p 10930 -r /storage/emulated/0 -n "Гостиная"
```

Параметры (также через переменные окружения MEDIA_HOST/MEDIA_PORT/MEDIA_ROOT/MEDIA_NAME/MEDIA_BACKGROUND):

| Флаг | Описание | По умолчанию |
|------|----------|--------------|
| `-H, --host` | адрес прослушивания | `0.0.0.0` |
| `-p, --port` | порт (TCP и UDP) | `10930` |
| `-r, --root` | корневая папка | `/storage/emulated/0` |
| `-n, --name` | имя сервера для автообнаружения | `media-server` |
| `-b, --background` | фоновый режим (без консоли) | выкл. |

### Подтверждение доступа (защита)

При первом обращении нового устройства сервер спрашивает в консоли:

```
=== Запрос доступа ===
Устройство: "Samsung SM-T230"  (IP 192.168.0.55, id a1b2c3…)
Разрешить? [y/n] (30 сек):
```

- `y` — разрешить, `n` — отклонить.
- Если ответа нет 30 секунд — автоматический отказ.
- Разрешённые устройства сохраняются в `/data/adb/api_plaer/allowed.json` (id, имя, дата) и больше не спрашиваются.
- Идентификатор — ANDROID_ID устройства + его название (передаются приложением).
- В фоновом режиме (`-b`) пускаются только устройства из `allowed.json`, остальным — отказ.
- Эндпоинт `/health` открыт (нужен для автообнаружения); `/list`, `/search`, `/download` — только для одобренных.

Команды консоли во время работы: `list` — список разрешённых, `revoke <id|имя>` — отозвать, `help` — справка.

### API

- `GET /health` — проверка (открыто).
- `GET /list?path=<отн>` — содержимое папки (JSON).
- `GET /search?q=<текст>[&path=<отн>]` — рекурсивный поиск (до 500).
- `GET /download?path=<отн>` — отдача файла с поддержкой Range (перемотка/докачка).

Идентификатор устройства передаётся заголовками `X-Device-Id`/`X-Device-Name` либо query-параметрами `dev`/`dn` (для потока libVLC).

## Приложение

- На главном экране — IP и порт, кнопка **Найти серверы в сети** (автообнаружение по UDP), список недавних подключений.
- Проводник: поиск, сортировка по имени (естественная, Серия 2 раньше Серии 10) с выбором направления, потянуть вниз — обновить, отметка «✓» на просмотренных.
- Скачивание в папку Downloads с прогрессом в шторке и кнопкой Отмена; докачка при повторном запуске.
- Плеер: продолжение с места остановки, окно подготовки/буферизации, жесты (двойной тап ±10 сек / пауза, горизонтальное перетаскивание — перемотка, вертикальное справа — громкость до 200%), скорость, соотношение сторон, полноэкранный режим.
- Кнопки «предыдущая/следующая серия»; по окончании — окно со списком остальных серий и автопереходом с обратным отсчётом.
- Автопереподключение потока при обрыве (с той же позиции), поддержка медиа-кнопок (MediaSession).
- Настройки: светлая/тёмная тема.
