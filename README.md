# Планшет Плеер + media-server

Проект из двух частей:

1. **`app/`** — Android-приложение для старого планшета (Android 4.4.2 / API 19). Воспроизводит видео через **libVLC**, умеет перематывать, искать и скачивать файлы.
2. **`server/`** — лёгкий многопоточный HTTP-сервер на Rust для другого (рутованного) устройства. Раздаёт файлы из `/storage/emulated/0` по локальной сети.

Авторизации нет — рассчитано на доверенную локальную сеть.

---

## Сервер (`server/`)

### Сборка под aarch64 (Android)

```bash
cd server
cargo install cargo-ndk
cargo ndk -t arm64-v8a build --release
# бинарник: target/aarch64-linux-android/release/media-server
```

### Запуск (вручную на устройстве)

```bash
./media-server                       # 0.0.0.0:10930, корень /storage/emulated/0
./media-server -p 8080 -r /sdcard    # свой порт и корень
```

Опции: `-H/--host`, `-p/--port`, `-r/--root`, `-h/--help`.
Переменные окружения: `MEDIA_HOST`, `MEDIA_PORT`, `MEDIA_ROOT`.

Сервер **многопоточный**: пул воркеров по числу ядер (минимум 4), поэтому одновременная потоковая раздача видео и Range-запросы не блокируют друг друга.

### API

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/` или `/health` | Проверка — возвращает `media-server ok` |
| GET | `/list?path=REL` | Содержимое папки (JSON: имя, тип, размер) |
| GET | `/search?q=TEXT&path=REL` | Рекурсивный поиск по именам файлов (до 500 результатов) |
| GET | `/download?path=REL` | Скачивание/поток с поддержкой Range (перемотка) |

`path` — относительный путь от корня. Переход вверх (`..`) запрещён — защита от выхода за корень.

## Приложение (`app/`)

### Сборка

```bash
cd app
gradle assembleRelease --no-daemon --stacktrace
# APK: app/build/outputs/apk/release/app-release.apk
```

Release-сборка идёт с **R8** (сжатие кода и ресурсов) и подписывается **debug-ключом**, чтобы APK ставился без настройки keystore.

> **Важно:** libVLC 3.5.x тянет транзитивные AndroidX-зависимости, поэтому в `gradle.properties` включён `android.useAndroidX=true` (и `android.enableJetifier=false` — jetifier не нужен, наш код использует только классы фреймворка).

### Характеристики

- `minSdk 19` (Android 4.4.2), `targetSdk 22`, ABI `armeabi-v7a`.
- libVLC: `org.videolan.android:libvlc-all:3.5.1`.
- AGP 8.5.2, Gradle 8.7, JDK 17.

### Функции

- **Главный экран:** ввод IP и порта + **история подключений** (тап — быстрый повторный вход).
- **Обзор файлов:** папки/файлы, **глобальный поиск** по всем файлам.
- **Плеер (libVLC):**
  - Перемотка **жестом** — пропорционально длине свайпа, с показом целевого времени.
  - Отдельная кнопка **перемотки на 1:30** (+ кнопки ±10 сек).
  - **Скорость** воспроизведения (0.5x–2.0x).
  - **Соотношение сторон** (По размеру / 16:9 / 4:3 / Растянуть / Оригинал).
  - Экран **не гаснет** во время воспроизведения.
- **Скачивание** файлов в папку Загрузок планшета.

## CI (GitHub Actions)

`.github/workflows/build.yml` — две задачи:

| Задача | Что делает | Артефакт |
|--------|------------|----------|
| `server` | Кросс-компиляция media-server под aarch64 через cargo-ndk | `media-server-aarch64` |
| `app` | `assembleRelease` (R8 + подпись debug-ключом) | `tablet-player-apk` (`app-release.apk`) |

Запуск: при push в `main`/`master` и вручную (`workflow_dispatch`).
